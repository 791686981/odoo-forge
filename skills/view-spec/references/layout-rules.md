# Odoo 视图布局通用规则

> 画线框图前必读。从 Odoo 18 视图规范中提炼的布局约束，确保线框图对应真实可实现的 Odoo 页面。

---

## 1. Form 视图

### 结构层级

```
form
├── header              ← 全宽横条：操作按钮（左）+ 状态栏（右）
├── sheet               ← 主内容区，居中响应式，有左右留白
│   ├── button_box      ← 统计按钮区，浮动在右上角
│   ├── oe_title        ← 标题区（编号+名称），与 button_box 同行靠左
│   ├── group           ← 字段分栏容器
│   │   └── group       ← 可嵌套，实现左右分栏
│   ├── separator       ← 分隔线（可带标题文字）
│   ├── notebook        ← Tab 页签容器
│   │   └── page        ← 单个 Tab 页（可条件隐藏）
│   └── (其他 HTML)
├── chatter             ← 消息 + 活动 + 关注者，在 sheet 之后
└── footer              ← 仅弹窗场景用（target="new"）
```

### Header 区域

header 是 sheet 上方的全宽横条，放工作流按钮和状态栏：

```
┌── header ────────────────────────────────────────────────────┐
│ [按钮1] [按钮2] [按钮3]              ○ 状态1 → ○ → ● 状态N │
│ ← 操作按钮，靠左                      ← statusbar，靠右 →   │
└──────────────────────────────────────────────────────────────┘
```

- **按钮在左**，从左到右排列。不同状态下显示不同按钮（通过 `invisible` 控制）
- **状态栏在右**，用 `statusbar` widget 渲染，显示状态流转路径
- 如果没有工作流（如配置表单），header 可以省略

### Sheet 顶部：button_box + oe_title

button_box 和 oe_title 在 sheet 最上方**并排显示**：

```
┌── sheet ─────────────────────────────────────────────────────┐
│                                    ┌── button_box ─────────┐ │
│  ┌── oe_title ──────────────┐     │ [📊 12 任务] [💰 ¥50K]│ │
│  │  编号    PRJ-001         │     │ [📎 3 附件]           │ │
│  │  名称    XXXXX（大字 h1）│     └────────────────────────┘ │
│  └──────────────────────────┘                                │
│  ...                                                         │
└──────────────────────────────────────────────────────────────┘
```

- **button_box 浮动在右上角**，放 3-5 个统计按钮（`oe_stat_button` + `statinfo` widget）
- **oe_title 在左上角**，用 `<h1>` 包裹主标题（通常是名称字段），编号可在上方小字显示
- 两者视觉上同一行，button_box 在右、oe_title 在左
- 不是所有表单都有 button_box（简单表单可以省略）
- 不是所有表单都有 oe_title（简单表单可以直接把名称放 group 里）
- 注意：部分官方模块（如销售订单）会把统计按钮放进控制面板而非 sheet 内，这是特殊处理，**自定义视图按 sheet 内右上角画**

### Group 列布局

这是 Form 视图最核心的布局机制：

- **默认 2 列**，可通过 `col="N"` 自定义（如 `col="4"` 四列、`col="12"` 十二列网格）
- 字段在 group 内**自动带 label**：label 占 1 格 + 字段占 1 格 = 每个字段实际占 2 格
- `colspan="N"` 让子元素跨多列
- `newline` 强制换行，从下一行开始（不填满当前行剩余列）
- **group 可带标题**：`string="标题"` 会在 group 上方显示小标题
- **嵌套 group**：常用"外层 group 包两个内层 group"实现左右两栏

```
┌── group (外层, col=2) ──────────────────────────────────┐
│ ┌── group "基本信息"(左栏)┐  ┌── group "日期"(右栏)──┐ │
│ │ label  字段A            │  │ label  字段D          │ │
│ │ label  字段B            │  │ label  字段E          │ │
│ │ label  字段C            │  │ label  字段F          │ │
│ └─────────────────────────┘  └───────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

**字段在 group 外部**的行为不同：不自动带 label，需要手动用 `<label>` 元素补上。

### Notebook 页签

notebook 放在 group 之后，承载明细行和次要信息：

- **notebook 不能放在 group 内部**，它和 group 是 sheet 的同级子元素
- 每个 `page` 有标题（`string` 属性），显示为 tab 名
- page 可以通过 `invisible` 条件隐藏（比如某状态下隐藏某 tab）
- **第一个 tab** 通常放 One2many 明细行
- 后续 tab 放：其他信息、备注、审批记录等

### Chatter 区域

chatter 在 sheet 之后，包含三个部分：

```
├── chatter ───────────────────────────────────────────────┤
│  [发送消息] [备注] [活动]              🔍 📎 👥 关注者   │
│                                                          │
│  消息列表（时间线）                                       │
│  ...                                                     │
└──────────────────────────────────────────────────────────┘
```

- **消息线程**（message_ids）：沟通记录、系统日志
- **活动**（activity_ids）：待办活动（如"2天后跟进"）
- **关注者**（message_follower_ids）：订阅此记录的人
- 前提：模型需要继承 `mail.thread`
- 不是所有表单都需要 chatter（配置类表单通常不加）

### 关系字段的内嵌视图

One2many / Many2many 字段可以在表单内嵌入子视图：

- 默认以 **inline list** 显示（表格形式，可行内编辑）
- 也可以内嵌 form、kanban 等（通过 `mode` 属性切换）
- 内嵌视图功能受限：没有独立搜索栏，行数有分页限制（默认 40 行）
- 画线框图时，One2many 字段画成一个嵌在 tab 里的小表格

### 弹窗表单 vs 普通表单

弹窗表单（通过 `target="new"` 的 action 打开）与普通表单的区别：

| | 普通表单 | 弹窗表单 |
|--|---------|---------|
| header | 有（按钮+状态栏） | 可有可无 |
| sheet | 有 | 有 |
| footer | 无 | 有（放确定/取消按钮） |
| chatter | 常有 | 通常无 |
| 整体感觉 | 完整页面 | 紧凑对话框，字段更少 |

### 关键约束汇总

- **notebook 不能放在 group 内部**
- **chatter 在 sheet 之后**，不在 sheet 内部
- **button_box 浮动右上角**，和 oe_title 同行
- **footer 仅弹窗用**，普通表单不画 footer
- 同一字段可出现多次，但 `required` 属性必须一致

### CSS 定位类

这些类影响字段的视觉位置，画线框图时需要知道：

| 类名 | 效果 |
|------|------|
| `oe_inline` | 字段不另起一行，紧跟前一个元素（常用于日期+时间并排） |
| `oe_left` / `oe_right` | 浮动到左/右侧（常用于图片字段） |
| `oe_avatar` | 头像样式，约 90×90px，浮动在左上角 |
| `oe_read_only` / `oe_edit_only` | 仅在查看/编辑模式显示 |
| `oe_stat_button` | 统计按钮样式，用在 button_box 内 |

## 2. List 视图

### 结构

```
控制面板（选中记录后，中部出现批量操作）:
  [新建] [上传]  |  9已选取 × [创建发票] [打印] [操作]  |  1-9/9 < >  视图切换

┌── 列头 ──────────────────────────────────────────────────────────┐
│ ☐ │ 名称/编号  │ 日期        │ 客户        │ 负责人 │活动│总计│状态│⇅│
├──────────────────────────────────────────────────────────────────┤
│ ☑ │ S00022    │ 2026-04-05  │ Acme Corp   │[👤]    │ ⊙ │$53,000│销售订单│
│ ☑ │ S00021    │ 2026-04-05  │ Acme Corp   │[👤]    │ ⊙ │$17,250│销售订单│
│ ☑ │ S00007    │ 2026-04-05  │ Gemini Furn │[👤]    │✓ Check..│$1,706│销售订单│
│ ☐ │ S00003    │ 2026-04-05  │ Ready Mat   │[👤]    │✉ Answer..│$378│  报价单 │
├── 汇总行 ────────────────────────────────────────────────────────┤
│                                                    $82,958.50    │
└──────────────────────────────────────────────────────────────────┘
```

### 布局规则

- 列按**声明顺序从左到右**排列，第一列通常是主标识（名称/编号）
- **批量操作按钮**：选中记录后出现在控制面板中部，不在列表内部；`display="always"` 则始终显示
- **活动列**（`activity_ids`）：显示活动图标 + 文字，颜色区分紧急程度（绿/黄/红）；`⊙` 表示无活动
- **状态列**放最右（倒数第二，最右是 `⇅` 列设置），用**彩色 badge** 显示，不同状态不同颜色
- **`⇅` 列设置按钮**：列头最右侧，点击可切换 optional 列的显示
- **汇总行**：有 `sum` 聚合字段时，底部显示合计，靠右对齐
- **分组模式**：分组头显示分组值 + 记录数 + 聚合小计，可折叠展开（`▼/▶`）
- **control 区域**：仅用于 x2many 内嵌列表底部，放"添加一行"按钮

### 状态 Badge 画法

List 视图的状态不用 `○/●`，用彩色 badge（截图里是圆角填色标签）：

```
│ 销售订单 │   ← 绿色 badge
│  报价单  │   ← 蓝/灰 badge
│已发送报价│   ← 另一种颜色 badge
```

画线框图时用 `[状态名]` 方括号表示 badge 即可，不必区分颜色。

### 注意事项

- **数值聚合列避免放最后一列**：分组模式下最右列会被挤压
- `column_invisible` 整列隐藏，`invisible` 按行条件隐藏单个单元格
- `nolabel` 的列没有表头文字且不可排序，画线框图时表头留空

## 3. Kanban 视图

### 结构

```
┌── header（按钮，display="always"）────────────────────┐
│  [操作按钮]                                           │
├── 列（按 group_by 字段分列）──────────────────────────┤
│ ┌─ 阶段1 ──────┐  ┌─ 阶段2 ──────┐  ┌─ 阶段3 ──────┐│
│ │██████░░ N条  │  │ ████░░░ N条  │  │██████████ N条││  ← progressbar（可选）
│ │              │  │              │  │              ││
│ │┌── 卡片 ───┐ │  │┌── 卡片 ───┐│  │              ││
│ ││ 标题       │ │  ││ 标题       ││  │              ││
│ ││ 字段1      │ │  ││ 字段1      ││  │              ││
│ ││ 字段2      │ │  │└───────────┘│  │              ││
│ ││────────────│ │  │              │  │              ││
│ ││[👤] ¥金额  │ │  │              │  │              ││  ← footer（固定底部）
│ │└────────────┘ │  │              │  │              ││
│ └───────────────┘  └──────────────┘  └──────────────┘│
└───────────────────────────────────────────────────────┘
```

### 布局规则

- **最多 10 列**可见，超出折叠但可点击展开
- **卡片内部结构**：
  - 主体区（标题 + 关键字段）
  - footer 区（固定在卡片底部：负责人头像、金额、活动状态等）
  - aside 区（可选，卡片左侧窄条，常放图片）
- **progressbar**：在列头显示，彩色段表示该列内记录按状态的分布
- **highlight_color**：卡片左边框颜色标识，用于快速分类识别
- **quick_create**：点击列头 `+` 在列底部弹出快速创建表单

### 卡片内容选取原则

卡片空间有限，只放 3-5 个最关键的信息：
- 标题（名称/编号）— 必有
- 1-3 个关键业务字段
- footer：负责人头像 + 关键数值（金额/进度/截止日期）

## 4. Search 视图

### 结构

```
search
├── field       ← 搜索字段（输入关键词匹配）
├── filter      ← 预设筛选器（点击激活）
├── separator   ← 分隔符（改变相邻 filter 的组合逻辑）
├── group       ← 逻辑分组（同 separator 效果）
└── searchpanel ← 左侧筛选面板（可选，附在 list/kanban 左侧）
```

### Filter 逻辑（重要）

```
不加 separator：
  [我的记录] [待审批] [已完成]        → 三个互为 OR
  选了"待审批"或"已完成"都能出来

加了 separator（用 --- 表示）：
  [我的记录] [待审批] --- [本月] [本季度]
  └──── 组1: OR ────┘     └── 组2: OR ──┘
        └──────── 组1 AND 组2 ──────────┘
  选了"待审批" + "本月" → 同时满足两个条件才出来
```

- **无 separator** → 相邻 filter 之间是 **OR**（满足其一即可）
- **有 separator** → 两侧的组之间是 **AND**（同时满足）
- 这是 Odoo 里最容易搞混的逻辑，画线框图时要标注清楚

### Searchpanel

- 显示在视图**左侧**，作为侧边栏（仅限 list/kanban 视图）
- `select="one"`：单选，`○/●` 单选按钮，支持层级树展开
- `select="multi"`：多选，`☐/☑` 勾选框，旁边显示记录数
- 默认最多显示 200 个值，超出截断

## 5. Settings 视图

### 结构

```
┌── 侧边栏 ──┬── 设置内容 ──────────────────────────────────┐
│            │                                              │
│  ● 应用1   │  ┌── setting (type=header) ───────────────┐ │  ← 全宽横幅
│  ○ 应用2   │  │  当前网站 / 当前公司                    │ │
│  ○ 应用3   │  └────────────────────────────────────────┘ │
│            │                                              │
│            │  ┌── block: 分组标题 ────────────────────┐  │
│            │  │                                       │  │
│            │  │  [✓] 开关标签           ← Boolean 左侧│  │
│            │  │      说明文字                          │  │
│            │  │      ┌── 子配置 ──────────────┐       │  │
│            │  │      │ 配置项   [输入框]       │       │  │
│            │  │      └────────────────────────┘       │  │
│            │  │                                       │  │
│            │  │  配置标签  [输入框/下拉] ← 非Boolean右│  │
│            │  │      说明文字                          │  │
│            │  └───────────────────────────────────────┘  │
└────────────┴──────────────────────────────────────────────┘
```

### 布局规则

- **Boolean 字段** → 左侧显示为开关 `[✓]/[ ]`，开启后可展开子配置
- **非 Boolean 字段** → 右侧显示为输入框/下拉，标签在上方
- `type="header"` 的 setting → 全宽横幅，用作范围分隔（"当前网站"/"当前公司"）
- `company_specific="1"` → 设置项旁显示公司图标，表示各公司可独立配置
- `documentation` → 问号图标，点击跳转到帮助文档

## 6. 常用 Widget 的视觉形态

Widget 决定字段在页面上长什么样，画线框图时需要用对应的视觉表达。

### 文本 / 基础

| Widget | 视觉形态 | 线框图画法 | 常见位置 |
|--------|----------|-----------|----------|
| `char` | 单行文本框 | `[_______________]` | group 内普通字段 |
| `text` | 多行文本域 | `[_______________]` × 多行 | 备注 / 描述字段 |
| `email` | 邮件输入（可点击发送） | `user@example.com` | 联系信息 |
| `phone` | 电话输入（可点击拨打） | `+86-138-xxxx-xxxx` | 联系信息 |
| `url` | 链接（可点击跳转） | `https://example.com` | 网站字段 |
| `binary` | 文件上传 / 下载 | `[📎 文件名.pdf  ✕]` | 附件字段 |
| `image` | 图片展示 | `[🖼 图片]` | 产品图 / 记录图片 |
| `image` + `oe_avatar` | 方形头像（~90×90） | `[👤]` | sheet 左上角、卡片 |
| `link_button` | 带图标的跳转链接 | `[🔗 查看详情]` | group 内只读关联 |
| `handle` | 拖拽排序手柄 | `≡` | list 明细行最左列 |

### 数值 / 统计

| Widget | 视觉形态 | 线框图画法 | 常见位置 |
|--------|----------|-----------|----------|
| `integer` | 整数输入框 | `[______123]` | 数量 / 计数字段 |
| `float` | 小数输入框 | `[_____1.50]` | 单价 / 小数字段 |
| `float_time` | 时长（小时:分钟格式） | `[__1:30]` | 工时 / 时长字段 |
| `monetary` | 货币金额（带币种符号） | `¥ [___1,000.00]` | 金额字段 |
| `statinfo` | 数字 + 标签（统计按钮） | `[📊 12 任务]` | button_box 内 |
| `progressbar` | 彩色进度条 | `██████░░░░ 60%` | kanban 列头、list 进度列 |
| `percentpie` | 圆形进度饼图 | `◕ 75%` | button_box 内、卡片 |
| `priority` | 星级评分（0-3星） | `★★☆` | 字段区域或卡片右上角 |
| `dashboard_graph` | 迷你趋势折线图 | `[📈 ▁▂▄▇▅]` | kanban 卡片内 |

### 日期 / 时间

| Widget | 视觉形态 | 线框图画法 | 常见位置 |
|--------|----------|-----------|----------|
| `date` | 日期选择器 | `[2026-03-01 📅]` | 日期字段 |
| `datetime` | 日期时间选择器 | `[2026-03-01 09:00 📅]` | 时间戳字段 |
| `daterange` | 日期范围（起 → 止） | `[2026-03-01] → [2026-12-31]` | 有效期 / 周期字段 |
| `remaining_days` | 剩余天数倒计时徽标 | `[🔴 逾期3天]` / `[🟡 还剩5天]` / `[🟢 还剩30天]` | 截止日期字段 |

### 选择 / 状态

| Widget | 视觉形态 | 线框图画法 | 常见位置 |
|--------|----------|-----------|----------|
| `selection` | 下拉选择框 | `[选项A ▼]` | 类型 / 状态下拉 |
| `radio` | 平铺单选按钮 | `(●) 选项A  (○) 选项B  (○) 选项C` | 选项少（≤4）的选择字段 |
| `selection_badge` | 横排可点选徽标 | `[ 选项A ]  [●选项B]  [ 选项C ]` | 可见选项少的状态切换 |
| `badge` | 只读彩色徽标 | `[状态名]` | list 状态列、只读状态字段 |
| `statusbar` | header 内水平状态流 | `○ 草稿 → ○ 审批中 → ● 已通过` | header 右侧 |
| `boolean_toggle` | 滑动开关 | `●━━━` (开) / `━━━○` (关) | Settings、list 开关列 |
| `boolean_favorite` | 收藏星（单星切换） | `☆` / `★` | list 收藏列、字段区域 |

### 关联字段

| Widget | 视觉形态 | 线框图画法 | 常见位置 |
|--------|----------|-----------|----------|

## 7. 通用布局原则

1. **信息密度递减** — 从上到下，信息重要性递减。最关键的字段放最上面。
2. **左主右辅** — 左侧放主要信息（名称、客户、类型），右侧放辅助信息（日期、金额、负责人）。
3. **相关字段分组** — 同一业务含义的字段放在同一个 group 或 tab 里。
4. **明细放 Tab** — One2many 明细行放 notebook 的第一个 tab。
5. **操作在顶部** — 工作流按钮放 header，不散在 sheet 里。
6. **状态栏靠右** — statusbar widget 在 header 右侧。
7. **弹窗更紧凑** — 弹窗表单（target=new）用 footer 放按钮，通常不加 chatter，字段更少。
8. **内嵌视图受限** — x2many inline list/form 功能受限：不支持独立的 search 视图，行数有分页限制。
