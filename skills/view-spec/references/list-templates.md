# List 视图规格模板

## 选择骨架

| 场景 | 骨架 | 判断依据 |
|------|------|----------|
| 基础列表，只看数据 | 简单 List | 纯展示，点击行进 Form |
| 有批量操作 | 标准 List | 需要选中多条批量处理 |
| 有分组聚合 | 分组 List | 按维度聚合查看 |

## 配套说明格式

线框图之后，附列说明表：

```markdown
**列说明**

| 列 | Widget | 备注 |
|----|--------|------|
| 列标签 | widget名 | 排序方式 / 聚合方式 / optional 说明 |
```

**常用列 widget：**

| 情形 | Widget |
|------|--------|
| 文本/编号 | `char` |
| 日期 | `date` |
| 关联对象 | `many2one` |
| 关联对象+头像 | `many2one_avatar`（只显示头像）|
| 金额（含列汇总）| `monetary`，备注"列底部显示合计" |
| 状态（彩色标签）| `badge` |
| 活动列 | `activity_ids` |
| 可选列（默认隐藏）| 任意 widget，备注"optional，默认隐藏" |

**列设计规则：**
- 4-8 列为宜，超过 8 列考虑 optional
- 第一列：名称或编号（主识别字段）
- 最后一列：状态（`badge`）
- 倒数第二列：`⇅` 列设置按钮（框架自动生成，不需要画）
- 金额列放靠右，方便对齐
- 默认排序标注在备注中

## 填充示例

需求："合同列表，按部门分组，显示合同金额汇总"

```
┌── 列头 ──────────────────────────────────────────────────────────┐
│ ▼ 工程部（3 条）                                  小计: ¥320,000 │
│   ☐ │ 合同编号     │ 合同名称     │ 客户      │ 金额    │ 状态  │ │
│   ──┼──────────────┼──────────────┼───────────┼─────────┼───────│ │
│   ☐ │ HT-2026-001 │ XXXXX        │ XXXXX     │¥120,000 │[执行中]│ │
│   ☐ │ HT-2026-003 │ XXXXX        │ XXXXX     │ ¥80,000 │[执行中]│ │
│   ☐ │ HT-2026-005 │ XXXXX        │ XXXXX     │¥120,000 │[草稿] │ │
│                                                                  │
│ ▶ 设计部（2 条）                                  小计: ¥150,000 │
│ ▶ 采购部（4 条）                                  小计: ¥480,000 │
├── 总计 ──────────────────────────────────────────────────────────┤
│                                               总计: ¥950,000     │
└──────────────────────────────────────────────────────────────────┘
```

**列说明**

| 列 | Widget | 备注 |
|----|--------|------|
| 合同编号 | `char` | 默认降序排列 |
| 合同名称 | `char` | — |
| 客户 | `many2one` | — |
| 金额 | `monetary` | 列底部显示合计；分组头显示小计 |
| 状态 | `badge` | 草稿=灰，执行中=绿，已完成=蓝，已取消=红 |
