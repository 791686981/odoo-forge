# 视图规格：设备报修工单（maintenance.request）

---

## 视图组合

| 视图类型 | 说明 |
|----------|------|
| Form | 报修详情，含工作流 + 维修用料明细 |
| List | 工单列表，查看全部工单状态和进度 |
| Kanban | 工单看板，按处理阶段分列拖拽 |
| Search | 搜索 + 筛选 + 分组 |

---

**字段说明**

| 字段 | Widget | 备注 |
|------|--------|------|
| 工单编号 | `char` | 序号自动生成，不可编辑 |
| 报修标题 | `char` | 必填，oe_title 大字显示 |
| 报修设备 | `many2one` | 关联 maintenance.equipment，必填 |
| 故障类别 | `selection` | 可选值：机械故障 / 电气故障 / 管道泄漏 / 其他 |
| 报修人 | `many2one_avatar` | 默认当前用户 |
| 紧急程度 | `priority` | 0-3 星，默认 0 |
| 报修日期 | `date` | 默认今天 |
| 计划维修 | `date` | 选填，维修中状态才可编辑 |
| 完成日期 | `date` | 只读，完成时自动填入 |
| 维修负责人 | `many2one_avatar` | 关联 res.users |
| 故障描述 | `text` | 多行文本，放在 group 下方单独一行 |
| 明细行 > 物料名称 | `many2one` | 关联 product.product |
| 明细行 > 数量 | `float` | 默认 1 |
| 明细行 > 单价 | `monetary` | — |
| 明细行 > 小计 | `monetary` | 只读，数量 × 单价 |
| 维修总费用 | `monetary` | 只读，汇总明细行小计 |

**操作按钮**

| 按钮 | 显示条件 | 行为 |
|------|----------|------|
| [提交报修] | 草稿状态 | 状态→已报修，通知维修负责人 |
| [开始维修] | 已报修，且当前用户是维修负责人 | 状态→维修中 |
| [完成] | 维修中 | 状态→已完成，自动填入完成日期 |
| [取消] | 非已完成状态 | 状态→已取消 |

**状态栏**：statusbar 显示 `草稿 → 已报修 → 维修中 → 已完成`。已取消不显示在状态栏。

## Kanban 视图

**卡片线框图**

```
┌──────────────────────────┐
│ ★★☆  MR/2026/003        │
│ 空压机异响需检修          │
│ 设备: 3号空压机           │

└──────────────────────────┘
```

**卡片字段说明**

| 字段 | Widget | 位置 | 备注 |
|------|--------|------|------|
| 紧急程度 | `priority` | 主体（标题行左侧）| 星级 |
| 工单编号 | `char` | 主体（标题行右侧）| — |
| 报修标题 | `char` | 主体 | 加粗，主识别信息 |
| 报修设备 | `many2one` | 主体 | — |
| 报修日期 | `date` | 主体 | — |
| 报修人 | `many2one_avatar` | footer | 显示头像 + 姓名 |

**列/阶段说明**

分组字段：`state`，按以下值分列：

| 列名 | 含义 |
|------|------|
| 草稿 | 新建，未提交 |
| 已报修 | 已提交，等待维修人员接单 |
| 维修中 | 维修人员正在处理 |
| 已完成 | 维修完毕 |

- 卡片可拖拽在列间移动
- 已取消不作为列显示（异常态）

## Search 视图

```
🔍 [工单编号 / 报修标题 / 设备名称                  ]            [▲]

  ┌── 筛选 ──────────────┬── 分组方式 ──────────┬── 收藏夹 ──────────┐
  │   我报修的           │  按设备              │  保存当前搜索 ▼    │
  │   我负责的           │  按故障类别           │                    │
  │ ──────────────────   │  按负责人            │                    │
  │   待处理             │  按月份              │                    │
  │   维修中             │                      │                    │
  │   已完成             │                      │                    │
  │ ──────────────────   │                      │                    │
  │   本月               │                      │                    │
  │   本季度             │                      │                    │
  └──────────────────────┴──────────────────────┴────────────────────┘
```

**搜索字段**：输入关键词时匹配以下字段：
- 工单编号（name）
- 报修标题（title）
- 设备名称（equipment_id.name）

**筛选器**

| 标签 | 过滤条件 | 所在组 |
|------|---------|-------|
| 我报修的 | 当前用户是报修人 | 组1 |
| 我负责的 | 当前用户是维修负责人 | 组1 |
| 待处理 | 状态=已报修 | 组2 |
| 维修中 | 状态=维修中 | 组2 |
| 已完成 | 状态=已完成 | 组2 |
| 本月 | 报修日期在本月内 | 组3 |
| 本季度 | 报修日期在本季度内 | 组3 |

> 组1 内 OR；组2 内 OR；组3 内 OR；三组之间 AND。

**分组依据**

| 标签 | 字段 |
|------|------|
| 按设备 | equipment_id |
| 按故障类别 | category |
| 按负责人 | assigned_to |
| 按月份 | date_reported:month |

## 菜单结构

```
设备管理
└── 报修工单
    ├── 我报修的       → 过滤当前用户是报修人，默认 List
    ├── 待处理工单     → 过滤状态=已报修，默认 Kanban
    ├── 维修排期       → 默认 Calendar（仅维修人员+管理员可见）
    └── 全部工单       → 无过滤，默认 List（仅设备管理员可见）
```

## Pivot 视图

**默认布局：**
- 行：报修设备（equipment_id）
- 列：报修日期按月（date_reported:month）
- 度量：维修总费用（total_cost, sum）

**度量字段**

| 度量 | 字段 | 聚合 | 说明 |
|------|------|------|------|
| 维修总费用 | total_cost | sum | 默认显示 |
| 工单数 | __count | count | — |

**分组维度**

| 维度 | 字段 | 用法 |
|------|------|------|
| 设备 | equipment_id | 行分组（默认） |
| 故障类别 | category | 行分组 |
| 报修日期 | date_reported | 列分组（按月，默认） |
| 维修负责人 | assigned_to | 行分组 |

---

## Graph 视图

**默认图表：** 柱状图

**字段配置**

| 字段 | 角色 | 说明 |
|------|------|------|
| date_reported | X 轴（按月分组） | 时间维度 |
| total_cost | Y 轴（度量，sum） | 费用汇总 |
| category | 分组（颜色区分） | 按故障类别分色 |

**推荐图表组合：**
- 柱状图：按月统计维修费用
- 饼图：故障类别分布占比
- 折线图：月度维修数量趋势
