# 阶段 Gate 规则

## BRD -> Solution

### 进入条件

- 业务目标已清楚
- 核心角色和主流程已成型
- 关键规则和异常流已具备基本闭环
- 没有决定性缺失会直接阻断方案设计

### 红旗

- 主流程都还没理清
- 关键角色或对象缺失
- 大量问题还停留在表面诉求

## Solution -> PRD

### 进入条件

- 方案方向已基本收敛
- 模块与能力蓝图清楚
- 关键取舍和 GAP 已有结论
- 没有关键路径上的方案未决项

### 红旗

- 方案仍停留在大量并列候选
- 官方 / OCA / 定制边界没说清
- 是否重定制都还没定

## PRD -> TRD

### 进入条件

- 规则、状态、权限、视图和异常流已经足够清楚
- TRD 不需要回头猜业务逻辑
- 关键待确认项不再阻断技术设计

### 红旗

- 状态和规则未定
- 关键交互未定
- 权限边界不清

## TRD -> DEV

### 进入条件

- 模块、模型、权限、视图实现思路清楚
- 技术前提明确
- 开发可以直接开工

### 红旗

- 核心模型或行为定义缺失
- 权限 / 视图实现冲突
- 关键依赖或实现边界未定

## UAT -> 执行验收

### 进入条件

- 主路径和关键规则均有验收步骤
- 验收步骤可执行
- 预期结果明确

### 红旗

- 只有功能清单，没有操作闭环
- 关键异常流没有验收
- 非研发人员无法照着执行
