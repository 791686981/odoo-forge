# PRD 输出模板

使用本模板组织并写入 Notion 中当前 EPIC 页面下的 `PRD` 页面。

这份模板是仓库内的写作辅助，不是最终落地文档。
最终权威内容必须写回当前 EPIC 页面下的 `PRD` 页面。

原则：

- 只写已经确认的产品规格
- 明确承接域级 `BRD` 和 `Solution`
- `Solution` 中影响本 EPIC 的 GAP 和方案边界要显式体现在 `PRD`
- 未决问题统一进入当前业务域根页面下的页面 `待确认事项`
- 需要暂时保留但尚未验证的判断写入 `关键假设`
- 被明确放弃的路径写入 `已放弃 / 暂缓方案`
- 用户故事继续使用稳定编号 `US-01`、`US-02`……

---
