# 1 组织定位与核心创值功能

组织定位：聚焦能源化工、燃气行业，以数智化为支撑，提升设计牵引的低碳数智一体化服务能力，围绕客户需求，持续提升设计创值能力，支撑 EPC 一体化业务发展。

# 核心创值功能：

快速响应客户需求，发挥专业统筹，打造以规划设计牵引的 EPC 一体化服务能力，落地关键创值事项；完善规划设计专业体系搭建与机制建设，推动设计服务能力与数字化设计能力打造，落地产品、服务和关键能力的沉淀和创值验证；聚合设计生态资源，统筹资源调配，推动设计人才队伍建设与能力提升。

# 6 人力资源管理要求

# 6.1 人员招聘与调配

# 6.1.1 确定人力资源需求

各组织负责人根据人员编制及业务发展需要，制定本组织人力资源需求计划

提出申请，由综合智能能力群-人才激发汇总审核，经公司一号位审批后执行。

# 6.1.2 人员招聘

# 6.1.2.1 职责

（1）综合智能能力群-人才激发负责组织对应聘人员的面试、录用、试用、转

正等工作；

（2）各组织参与人员面试，并负责跟踪录用人员实习情况以及转正评定工作。

# 6.1.2.2 面试

应聘者首先需填写《应聘登记表》或自制简历交于招聘负责人，然后参加面

试。面试过程中，面试人员主要对应聘者的专业能力、技术专长、综合素质、工作

稳定性等方面进行评价分析，面试完毕后，相关面试人填写《面试测评单》。

# 6.1.2.3 试用

（1）综合智能能力群-人才激发负责安排新员工的试用，新员工试用期为三

个月；

（2）新录用的管理岗位人员及专业高级及以上人员，由综合智能能力群-人才激发负责组织编制实习计划并安排实习指导人。实习指导人和新员工每月底共同评价实习效果，并由新员工编写总结，每月提报至综合智能能力群-人才激发招聘负责人；

（3）新录用的非管理岗位人员及专业二级及以下人员，由所在实习组织制定实习计划并安排实习指导人。实习指导人和新员工每月底共同评价实习效果，并在试用期最后一个月编写总结，提报至综合智能能力群-人才激发招聘负责人。

# 6.1.2.4 转正

（1）新员工试用期结束前3天，由综合智能能力群-人才激发招聘负责人发起转正流程，新员工本人按要求上传相关材料，所在组织负责人、人才激发负责人、一号位审批；

（2）建立新员工人事档案。

# 6.1.3 人员调配

6.1.3.1 综合智能能力群-人才激发通过外部招聘、内部竞聘等方式补充公司岗位空缺；

6.1.3.2根据公司业务发展、年度任务等相关情况，综合智能能力群-人才激发牵头，按需对员工的工作岗位及所属组织做出调整并与员工本人沟通一致，经调入/调出组织负责人、一号位批准后执行。

# 6.2 培训

# 6.2.1 培训目标

# 6.2.1.1 通过培训教育使员工充分意识到：

（1）满足客户和法律法规要求的重要性；  
（2）违反这些要求所造成的后果；  
（3）自己从事的活动与公司发展的相关性；

# 6.2.1.2 通过培训教育使员工岗位的适应性和岗位技能增强。

# 6.2.2 培训计划和组织

（1）综合智能能力群-人才激发每年12月份制订公司次年年度培训计划，经公司一号位批准后，组织实施并考核其有效性；  
（2）每次培训时，参训人员应在《培训记录表》上签到或利用学习平台线上签到，并由培训负责人收集、保存；  
（3）新进人员报到后，综合智能能力群-人才激发协助各组织做好新员工入

司培训。

# 6.2.3 培训形式

# （1）内部培训

公司根据培训需求采取讨论、讲座、请专家来公司传授知识等方式进行培训另外公司还可通过轮岗、教练式训练等方式进行培训。

# （2）外部培训

公司外部培训机构组织的培训，包括公派外出培训、学历教育等形式。外部培训分为短期培训、中期培训(进修）、长期培训(学历教育）。需填写《外出培训申请表》。

# 6.2.4 培训内容

培训内容按照培训目的的不同具体可分为四类。

# 6.2.4.1 新进人员岗前培训内容：

（1）介绍公司质量方针、质量目标、工作环境；  
（2）介绍公司组织架构；  
（3）学习公司管理制度；  
（4）学习公司企业文化，促进员工的认同感。

# 6.2.4.2 新进人员岗位职责与技能培训内容：

（1）岗位职责培训，主要针对岗位的职责标准、岗位设置目的进行培训，使新进员工充分理解本岗位的工作要求以及工作能力；  
（2）岗位技能培训，通过岗位所需技能的培训和认证，确保新进人员能够胜任岗位工作。公司严格执行设计资格考试认证、SAP信息系统考试认证、技术工种的考试认证等，对于技能达不到要求的人员，公司将通过进一步的培训教育提升技能，若培训后仍达不到要求的，则进行调岗或辞退。

# 6.2.4.3 继续教育培训

（1）专业技术讲座；  
（2）标准、规范；  
（3）专业技术交流；  
（4）计算机软件应用。

# 6.2.4.4 特殊岗位培训

国家规定的必须持有资格证上岗的特殊培训，如:造价工程师、一级建造师、电气仪表工程师等相关注册工程师继续教育培训。

# 6.2.5 培训的有效性评估

培训的有效性评估按照《员工培训管理办法》的相关规定进行，内部培训的组织人员须在内部学习平台中创建培训，并要求参训人员完成培训课题的签到和评估，除此之外，还可视需要采取以下几种方式对培训项目进行效果评估：

（1）考试、考核：指书面考试，现场操作考核；  
（2）培训实践评估：由直接主管对受训员工的工作表现进行评估，连续评估三个月（该条款适用于对重要、关键培训项目的评估）。

以上培训评估记录报公司培训管理部门备案。

# 6.3 创值评估

（1）根据集团制度要求，各组织负责本组织员工的创值评估，一号位负责关键角色和各组织的创值评估。季度创值评估与年度创值评估相结合，及时把握员工的创值表现，并通过激励措施、教育培训等进一步提升员工的创值与能力。  
（2）通过个人创值情况的评估使各层级员工意识到自己的工作价值，并通过内部学习与成长，提升自身能力，使其更好支撑组织创值目标实现。  
（3）综合智能能力群-人才激发负责个人年终创值评估的计划制订、组织实施和反馈。个人年终创值评估成绩作为员工晋级、年终奖核发和薪酬调整的参考依据。

（4）综合智能能力群-人才激发每年年终对员工能力进行盘点和评估，重点评估关键角色和核心人才， 通过测评对关键角色和核心人才现有能力和未来潜力进行评定，确定关键角色和核心人才后备梯队，并通过培训教育进行培养， 提高公司人才队伍的整体水平，从而保障公司经营目标、质量目标的达成。  
（5）员工的晋升、薪酬调整将结合个人创值评估和能力评估综合评定。

# 6.4 设计资格管理

（2）各组织负责本组织设计人员资格的申报，对申请人聘用关系合规性、填报资料准确性和真实性进行把控，对设计人员资格合规应用。  
（3）技术质量智能能力群负责组织设计人员资格的考评及聘任工作，一号位负责资格聘任的审批。  
（4）技术质量智能能力群负责统一对各级设计人员的资格进行集中管理，每年组织资格考评及聘任工作。  
（5）设计人员的任职资格管理详见《工程设计资格管理办法》。

# 9 合同与投标管理要求

# 9.1 合同管理

9.1.1为正确评审和履行合同，对各类合同进行管理和监控，保障公司的合法权益

防范合同风险，结合公司实际，特制定《合同管理办法》。综合智能能力群-经营为

合同的归口管理部门，合同管理员负责合同管理工作。

9.1.2项目开发人员与业主谈判并与公司相关领导沟通确认后拟定合同初稿，合

同初稿经相关领导审核后，由经办人（市场支持人员或开发人）发起合同评审，合

同管理员组织合同评审。为确保评审流程合规性，合同执行线上评审。参与合同

评审的人员应签署明确、具体的意见，禁止使用“原则同意”、“基本可行”等

模糊性语言。

9.1.3合同评审前需做好前期调研工作，对于不符合合同主体资格要求或资信状

况不佳的相关公司和个人，不得与其签订合同。

9.1.4合同管理员根据合同的具体内容，从经济、法律、技术、财务等方面组织评

审人员进行评审。合同评审按照公司授权流程归口负责组织发布的授权管理相关

文件执行，授权文件以公布于公司平台的内容为准。

9.1.5合同经办人为合同评审第一责任人，负责跟催合同审核、会签进度，督促会

签人员在规定时间内进行评审，根据评审意见修改合同内容。合同管理员负责核稿，一号位负责审批。

9.1.6 合同管理员负责将评审后最终版合同形成纸质文件，并组织合同双方签字、盖章，以及后续台账的维护、正式合同的归档。经办人负责将合同管理员反馈的正式合同邮寄至相关客户，及合同平台电子版（PDF 格式）扫描件的上传。  
9.1.7 合同管理员负责协助处理合同的履行、变更和废除、合同的纠纷处理、合同文件的管理，按照《合同管理办法》制度执行。

# 11 设计过程管理要求

# 11.1 设计项目的策划

设计项目实施前应对设计过程进行策划，在确定设计过程各阶段和控制的策划时，应考虑，但不限于：

（1）设计活动的性质、持续时间和复杂程度；  
（2）所需的过程阶段，包括适用的评审；  
（3）所需的设计验证、确认活动；  
（4）设计过程涉及的职责和权限；  
（5）设计产品和服务所需的内部、外部资源；  
（6）设计过程参与人员之间接口的控制需求；  
（7）顾客及使用者参与设计和开发过程的需求；  
（8）对后续产品和服务提供的要求；  
（9）顾客和其他有关相关方所期望的对设计过程的控制水平；  
（10）证实已经满足设计要求所需的成文信息。

# 11.1.1 组建项目组

（1）项目任务下达人员根据接收到的设计委托书、设计合同或补充协议，在网络设计平台发布设计任务或下达《设计任务通知书》/《设计任务通知单》，由具有相应设计经理资格的人员承接对应的设计项目，担任项目设计经理。  
（2）设计经理负责对设计任务进行评审，必要时，可组建团队进行评审。经评审确认设计任务后，设计经理组建设计项目组。设计项目组任命的设、校、审等各级人员，应具备公司聘任的设计资格。  
（3）参加设计项目的各专业、各级设计人员按各自的职责开展设计工作。

# 11.1.2 设计开工策划

设计经理负责根据项目具体情况开展设计策划，按《设计开工报告编制规定》编制《设计开工报告》并召开开工会议，形成并保留会议签到记录。对于少于3 个专业参与的设计项目、项目建议书，可不编制《设计开工报告》，但应在《设计任务通知书》等策划文件中明确项目的设计策划，包括项目编制要求、人员安排等内容，且须在《专业设计任务委托书》中注明必要的条件/协作要求等。EPC 项目设计开工策划及实施，还应执行《EPC 项目设计管理办法》等 EPC 项目管理相关规定。

# 11.1.3 设计进度及专业协作进度安排

（1）设计项目进度按照《设计项目进度管理办法》执行。设计经理根据设计合同、设计任务委托书及公司内部下达的《设计任务通知书》或网络设计平台发布的设计任务等文件资料的要求编制总（装置、单元、分项）进度计划，并组织指导各专业负责人编制专业协作进度计划。  
（2）设计经理会同各专业负责人编制《设计进度计划表》，表中列出设计评审、专业间互提委托资料、设计输出等各阶段进度安排及完成时间，并报分管负责人进行签批。  
（3）项目设计按进度计划执行，当出现影响交付等进度偏差时，设计经理及时与设计委托方沟通，形成可追朔的沟通记录作为依据留存并对《设计进度计划表》进行调整升版。

11.1.4 公司可通过生产经营例会，设计经理可通过设计项目协调会定期检查设计计划的实施情况，协调项目实施过程中存在的问题。项目管理人员负责对各项目进度情况进行跟踪。

# 11.2 组织和技术接口

11.2.1 设计经理应组织落实并明确设计项目内外部接口，项目组按照《项目设计组织和技术接口规定》开展内外部接口的实施工作。  
11.2.2 设计经理应对参与项目工作的组织或专业之间的内部接口进行管理，包括明确职责分工（各专业分工和项目组成员管理工作的分工等）、各专业间互提设计资料、配合时间等。各专业之间互提资料以《专业设计任务委托书》的形式进行其内容和深度应能满足接受专业正常开展设计工作的要求。  
11.2.3 两个以上专业的各单元由单元负责人负责组织单元协调会，各专业互相交流各自设计意图，提前做好规划，避免错漏碰发生。单元协调会内容包括：

（1）上游专业条件是否有遗漏和更新；  
（2）下游专业条件落实是否有问题；  
（3）各专业空间位置是否有碰撞（管道、风管、电缆桥架、留洞位置、梁柱等）  
（4）门、窗、通道是否有其他专业设施遮挡等。

11.2.4 单元协调会形成会议纪要与签到记录，并均予以保存。  
11.2.5 联络单适用于对设计内容的解释说明或者设计问题的确认，是设计发图前的解释说明、设计调整。  
11.2.5.1联络单分为项目级联络单、专业级联络单。

（1）项目级联络单范围：重大方案调整如设计范围调整、设计内容调整、采购设备配置调整、涉及多个专业调整（超过 2 个专业相关方案调整）等。  
（2）除项目级联络单外，其余均为专业级联络单。

# 11.2.5.2 联络单的批准发放

（1）项目级联络单由设计经理编制，主专业专业总工审核，总工程师批准。

（2）专业级联络单由各级设计人员编制，各专业负责人审核，设计经理批准。  
（3）审批后的《设计联络单》由设计经理负责外发。

# 11.2.5.3联络单的管理执行联络单外发及归档管理的相关要求。

# 11.3 设计输入

11.3.1 设计输入要求文件按项目阶段由设计经理编制。涉及多专业项目时除设计经理编制外，各专业负责人可根据需要另外编制本专业设计输入要求文件，其中专业间的互提设计输入条件应按《设计条件管理规定》执行。  
11.3.2 设计输入应满足合同要求，不应相互冲突，且应完整清楚，设计项目组应考虑：

（1）合同或委托书；  
（2）产品要求的确定和评审的结果；  
（3）项目实施策划；  
（4）工程会审文件和上级主管部门的批复文件；  
（5）上阶段项目文件，以前类似设计和开发活动的信息；  
（6）法律法规、技术体制、技术政策、标准规范、概预算定额；  
（7）主机设备、材料和系统的技术资料（土建项目应包括计算书）；  
（8）现场查勘收集的资料；  
（9）顾客提供的文件和资料（土建项目应包括地质勘察报告）；

（10）技术规格书；  
（11）相关的部门、专业之间的技术接口资料、互提资料及其要求。

11.3.3 设计经理对外来的所有可作为设计输入（或中间输入）的书面资料组织进  
行评审，专业负责人对专业间互提的设计输入的书面资料组织进行评审。  
11.3.4 所有与设计输入相关的文件资料均予以保存。

# 11.4 设计控制

11.4.1 设计经理是设计项目总体质量的负责人，应对设计过程进行控制，以确保

（1）编制设计策划文件，规定拟获得的结果；  
（2）实施评审活动，以评价设计的结果满足要求的能力；  
（3）实施验证活动，以确保设计输出满足输入的要求；  
（4）实施确认活动，以确保形成的产品和服务能够满足规定的使用要求或预

期用途；

（5）针对评审、验证和确认过程中确定的问题采取必要措施；  
（6）保留这些活动的形成文件的信息。

# 11.4.2 设计评审

设计评审应执行《设计评审管理规定》及其相关机制要求。设计评审主要分

为：

（1）设计方案评审；  
（2）设计输入评审；  
（3）设计过程中必要节点或问题的评审；  
（4）设计输出评审。

# 11.4.2.1 设计方案评审

设计方案评审是指对设计总体方案的评审，其控制要求：

（1）总体方案的评审，一般应在现场查勘工作完成之后，设计文件编制之前

进行，由设计经理组织项目组和相关人员评审；

（2）设计方案评审一般应采取会议方式进行。会议由专业群召集人或专业总

工组织，评审组织人根据评审需求组织项目组和相关人员参加；

（3）方案评审内容、评审结果、评审意见修改及落实情况等应填写到《设计评

审记录》中，并予以保存。

# 11.4.2.2 设计输入评审

设计输入评审是指通过评价设计输入内容，确定设计条件是否满足开展设计

工作的要求的评审，评审中应考虑，但不限于：

（1）基础文件资料是否充分、适宜；  
（2）功能与性能要求是否明确；  
（3）依据的设计法规标准是否恰当；  
（4）顾客的要求是否明确、合理；  
（5）以往设计的可参考信息是否适用；  
（6）前一阶段输出的文件是否满足要求；  
（7）进度安排是否合理。

设计输入评审内容、意见及评审结果应填写到《设计输入评审表》、《设计任务通知书》等设计输入评审信息文件中，并予以保存。

# 11.4.2.3 设计过程中必要节点或问题的评审

大型综合工程、重大 EPC项目或采用新技术、新工艺等项目的总平面图、PFD、PID、危险区域划分图、HAZOP审查、SIL定级、3D模型审查等，应根据项目情况开展项目级评审。项目设计过程中的重要过程、必要节点或问题，根据项目需求随时开展设计评审。设计评审内容、结果及落实情况应填写到《设计评审记录》中，并予以保存。

# 11.4.2.4 设计输出评审

设计项目放行前，应根据项目情况开展设计输出评审工作，具体开展方式应满足《设计评审管理规定》的相关要求，设计输出评审应考虑但不限于：

（1）设计输出是否符合设计输入的要求；  
（2）为采购、施工和服务的运作提供的信息（图纸、设备规格书、材料表、施工说明）是否齐全；

（3）设计输出是否符合合同规定要求；  
（4）设计输出是否符合国家的有关法律、法规和标准要求；  
（5）设计评审结论意见是否落实；  
（6）各级签署是否齐全。

设计输出评审的意见、建议及评审意见的修改、落实情况等应填写到《设计评审记录》中，并予以保存。

# 11.4.3 设计验证

（1）在设计文件批准出版之前，应按项目策划的安排实施设计验证，以确保设计输出满足输入要求，验证工作由专业设计人进行。开展设计项目验证工作的人员应具备相应的设计资格，设计验证工作的实施应满足《设计验证规定》、《设计文件签署规定》和《工程设计资格管理办法》中“各级设计人员的职责”、《专

业会签要求和范围规定》等要求。

（2）项目设计验证应不少于自校、校核、审核三级验证，是否需进行审定、专业间会审等验证需根据项目情况和《设计文件签署规定》中的相关要求执行。  
（3）为保证设计输出满足设计输入的要求，开展设计验证工作的人员应逐级对设计文件进行验证，并在确认所提问题的意见、建议完全落实后，完成设计产品和《设计(文本)校审记录》等文件的确认签署（含专业间会签）。

# 11.4.4 设计确认

设计文件交付后或实施前，应由顾客或政府主管部门对设计文件进行确认，确认文件应予以保存。

# 14.4.4.1 设计确认应分阶段进行，通常包括：

（1）可行性研究报告的评估；  
（2）设计方案的审查或确认（总图方案、建筑方案、工艺方案等）；  
（3）初步设计的审批；  
（4）施工图设计的审查。

14.4.4.2 设计经理应参加或组织相关人员参加确认活动（必要时，可提请公司相关领导参加）。确认活动通常以会议形式开展，参加确认活动人员职责：

（1）汇报设计方案及主要技术措施；  
（2）回答并解释与会人员的提问；  
（3）对会审决议内容应明确表态；  
（4）及时将会审决议内容汇报公司相关领导。

14.4.4.3 设计会议纪要文件或批复文件即为设计确认的结论，是作为设计更改或下一阶段设计的依据，应予以保存。  
14.4.4.4 项目批准部门下达的批准文件或顾客确认记录，可作为项目建议书、规划、可研报告、初步设计等设计文件的确认。  
14.4.4.5 下列方式均可作为工艺包、施工图设计的确认方式

（1）工艺包/施工图文件交付后经顾客进行会审确认合格，即为该项目的设计确认；  
（2）工艺包/施工图文件交付后，对接收方或顾客及施工单位进行设计交底，以《设计交底记录》作为该项目的设计确认；  
（3）对无上级批文或工艺包/施工图会审确认的工程，工程项目建成后，在合同规定的条件下达到合同规定的各项指标，正常运行72小时后，即为该项目符合要求。

# 11.5 设计输出

11.5.1 设计输出应按阶段和专业分别编制项目文件，并应符合以下要求：

（1）满足设计输入的要求；  
（2）符合公司合格品的规定，对于后续服务的提供过程是充分的；  
（3）包括或引用监视和测量的要求，适当时，包括接收准则；  
（4）规定对于预期目的、安全和正确提供的产品和服务的基本特性。  
（5）EPC项目请购文件应符合《项目设备材料请购控制程序》等相关要求。

11.5.2 设计输出文件应符合以下要求：

（1）设计输出文件的格式按《设计文件格式规定》执行；  
（2）设计输出文件的签署按《设计文件签署规定》、《专业会签要求和范围规

定》执行，并严格遵守《工程质量禁令管理规定》、《工程设计质量红黄线管理规

定》，禁止：违反国家、行业相关法律法规、标准规范强制性条款；未执行设校审程

序；设计人员违规签署等。

（3）设计输出文件的编号标识按《设计文件及技术文件编号规定》执行；  
（4）设计文件的组成及设计深度符合各专业工作设计内容和深度要求；  
（5）对有设计计算要求的项目，应提供《计算书》。

# 11.5.3 设计文件放行

（1）设计文件放行前，以及各专业的过程产品在交付下游专业前均应按照项目策划的安排对设计产品进行评审，以确保其要求已得到满足。设计文件按设计阶段依据《设计评审管理规定》或《设计成品质量评定规定》通过设计评审或质量评定等方式进行放行评审。  
（2）施工图设计文件在放行前应进行设计输出评审，输出评审应执行《设计评审管理规定》的相关要求。  
（3）项目申请报告、初步设计等设计文件放行前应进行质量评定，质量评定应执行《设计成品质量评定规定》的相关要求。  
（4）设计变更文件放行前应对设计更改的内容进行授权，完成《设计补充、变更通知单》和需会签变更图的签批，即为完成设计变更文件的授权，授权人员应满足《设计文件签署规定》中“设计变更文件签署”的要求。

# 11.5.4 产品交付

11.5.4.1 经放行后的设计成品交档案管理人员，并按照《技术档案管理办法》进行归档与验收。  
11.5.4.2 设计文件的出版和发放。

（1）设计文件出版和发放执行《设计文件晒发管理办法》。  
（2）设计成品具备出版条件后，晒图人员出版。  
（3）晒图人员负责设计文件的盖章，具体按照《印章管理规定》执行。  
（4）设计成品交付顾客时，要求顾客在图纸发放记录上签字确认（当设计图

纸为快递发送时保存发送单），并将该文件反馈至设计经理。

# 11.5.5 设计成品文件和项目文件的存档

根据设计过程管理要求，设计输入、设计控制、设计输出等过程需保留的文

件资料和设计成品文件均应予以保存。设计成品文件（包括可编辑版和签字PDF

版）和项目文件的保存应按《技术档案管理办法》执行。

# 11.6 设计更改

11.6.1 设计文件的更改可发生在文件编制过程中、文件交付后和服务过程中，设

计文件更改的信息来源包括但不限于：

（1）设计会审纪要或批复文件中对文件更改的要求；  
（2）项目人员认为必要的更改要求；  
（3）法律、法规的更改要求；  
（4）顾客要求的更改等。

11.6.2 对任何更改应按规定进行适当的识别、评审和控制，以确保这些更改对满足要求不会产生不利影响。更改后的文件（包括图纸、概预算等）应予以标识，并保留下列方面形成文件的信息：

（1）设计和开发更改文件；  
（2）与设计更改相关的评审结果文件；  
（3）设计更改的授权文件（当由非设计原因造成的设计更改时，应获取相关的书面更改授权文件并保留）；  
（4）为防止不利影响而采取的措施。

11.6.3 当设计更改发生在设计产品已交付顾客或相关方后出现时，应采用设计变更的形式，并满足设计变更管理相关要求。设计变更在实施之前，应经设计经理的确定，由相关设计人员编制《设计补充、变更通知单》，并根据需求增加变更图等设计更改文件。但当变更内容需相关专业会签时，必须增加可实现会签的变更图等设计更改文件。设计变更应进行验证，并根据更改情况适时进行评审和确认。《设计补充、变更通知单》和变更图等设计更改文件应作为设计产品及时交付给顾客和相关方。

11.6.4 当设计更改发生在设计产品已通过放行，但还未交付顾客或相关方时，由设计经理提交相关说明反馈档案和出版等下游接口组织。设计产品的更改可按放行前的评审、验证和放行等设计过程实施管控。更改替换的作废产品和过程记录等文件按不合格品进行处理。  
11.6.5 设计经理主要负责一般设计变更的组织实施及重大设计变更的评审提请。  
11.6.6 设计变更分为重大设计变更及一般设计变更。  
11.6.6.1 满足以下条件之一的，为重大设计变更：

（1）超出合同规定的变更；  
（2）原始设计输入文件资料（设计基础资料、设计基础条件等）的变更；  
（3）重大设计原则方案的变更（如扩大建设规模、提高建设标准、改变总图布

# 置、改变工艺流程和重要设备型号等）；

（4）影响关键性计划进度变化达两周以上的变更；  
（5）对其他装置、在建工程或已建成工程产生影响的变更；  
（6）变更预估金额在30万元以上或预估金额虽未达到上述金额，但已经达到合同额的5%的变更。

# 11.6.6.2 除重大设计变更之外的设计变更为一般设计变更。

# 11.6.7 设计变更的控制

（1）可行性研究和初步设计按顾客或审批部门的审批意见进行回复或变更

（补充）后,应重新进行验证和确认活动；

（2）施工图设计文件中一般设计变更，由设计人员或现场设计代表填写《设

计补充、变更通知单》，涉及到多专业变更时，需经相关专业设计人员会签，并由设计经理确认发放。

（3）施工图设计文件中重大设计变更，由设计经理负责向综合智能能力群及分

管领导汇报，并组织设计中心级评审，通过评审确定变更方案的可行性，评估对

工程进度、费用等方面影响，并确定原设计文件的处置和新工作组的人员安排等

设计评审按《设计评审管理规定》的要求开展，必要时可邀请顾客参加。

（4）设计变更文件作为设计产品的一部分，经授权放行后，应按设计过程要

求，及时完成交付和存档等工作，必要时还应反馈相关审批部门和施工项目部。

（5）EPC 项目设计变更同时还应执行 EPC 项目管理相关规定。

# 11.7 设计服务

设计产品交付后对顾客的服务，按《设计服务管理规定》执行。设计服务一般包括：

（1）设计交底，根据工程项目总体进度情况和顾客安排，参与设计交底工作。设计经理组织设计人员拟定设计交底提纲，向实施单位进行设计交底。  
（2）现场技术服务，根据需要选派设计代表进行现场施工、调试、开车等技术服务；  
（3）竣工验收，根据合同规定或顾客要求，设计经理、专业群召集人或专业总工选派专业负责人或设计人员参加工程验收或竣工验收；  
（4）适时进行设计回访并做好设计总结。

# 12 不合格管理要求

# 12.1 不合格的来源

任何不符合产品检查基准及顾客要求而造成非预期的交付或服务的产品或行为均为不合格。

（1）设计产品放行后，设计质量检查过程中发现的质量不合格；  
（2）来自设计产品交付后的工程会审、施工单位及顾客反馈的质量不合格信息；  
（3）各种会议（包括管理评审会议）提出的问题；  
（4）各种检查、审核（质量体系内部审核、外部审核）提出的不符合报告；  
（5）来自于投诉的不合格等。

# 12.2 不合格的等级

设计产品的不合格按其对项目的影响程度划分两个等级：

# 12.2.1 一般不合格：

（1）文件、图纸中的错、漏、碰、缺；  
（2）基础资料内容、深度不符合规定要求；  
（3）专业间互提条件不符；  
（4）设计评审、设计验证发现某项不合格；

（5）设计确认中的局部不合格；  
（6）成品晒印、装订过程中打印错误、不清楚、缺页、倒置等。

# 12.2.2 严重不合格：

（1）严重违反国家建设方针、政策、法律；  
（2）不符合设计输入的功能及使用要求；  
（3）存在原则性问题，必须返工；  
（4）存在不安全因素或造成严重浪费；  
（5）计算错误、输入资料严重失误；

# 12.3 不合格品的控制

12.3.1应对不符合要求的产品或行为进行识别和控制，防止非预期的使用或交付  
12.3.2业务组织、各级质量审核人员及质量管理组织应在设计过程管理过程中控

制不合格品产生的因素，确保设计产品满足设计要求。

12.3.3业务生组织负责按不合格程度，安排人员主持或参加评审，按处置意见安

排人员返工或修改，必要时通知顾客。

12.3.4业务组织负责对供方提供的不合格品进行审核、把关和处置，当发现有不

合格品时，由接口组织负责通知供方进行修改，直至合格。

12.3.5设计成品文件在出版、装订、分发过程中产生的不合格品由综合智能能力群、文件出版管理人员进行识别和控制，发现不合格品应退回返工、及时纠正，并作好隔离和记录。  
12.3.6文件归档过程中发现的不合格品由档案管理人员识别和控制，发现不合格品应退回返工，并作好隔离和记录。

# 12.4 不合格品的处置

在产品交付之后或产品和服务提供期间发现的不合格，应按不合格品的要求进行处置。根据其性质及其对产品和服务符合性的影响采取如下处置方案：

（1）纠正；  
（2）隔离、限制或暂停对产品和服务的提供；  
（3）告知顾客。

# 12.4.1 不合格品的处置方法

（1）一般不合格品由设计经理、专业负责人组织原设校审人员，按不合格评审处置意见及《设计验证规定》修改、补充、完善。经过再校审成为合格品。  
（2）总工程师负责主持和审核严重不合格品的评审和处置。严重不合格品，按评审的处置意见，一般采用返工方式达到合格。

（3）文件出版、分发过程中出现的不合格品按出图管理相关规定，返工、修改达到合格。  
（4）返工、修改由设计人员根据需求按《设计评审管理规定》、《设计验证规定》等规定完成评审和验证。  
（5）工程竣工交付验收和投产使用后，收到顾客投诉或主动回访发现不合格时，由业务组织负责组织整改并通报技术质量智能能力群。  
（6）业务组织负责监督本组织设计项目的具体实施，同时根据不合格品的处理结果，及时通知有关部门，必要时通知顾客。

# 12.4.2 不合格品的标识和隔离

（1）出现不合格品，要及时记录并通知有关部门停止使用，必要时通知顾客。  
（2）已发送顾客的设计产品，要及时通知顾客、施工安装单位。  
（3）经返工的合格文件，应有区别于原文件的标识。如：升版、日期，并在设计说明中明显标识。  
（4）需要返工或未经修改的不合格文件，应采取措施防止非预期使用，不得放行交付，不得转入其它专业，也不得归档。因法律原因需要保存的，应作出不合格标识，隔离存放。

（5）对设计成品交付后发现的不合格品，经修改或返工并经评审合格后，应及时交付给顾客，并对不合格品的作废通知顾客。

# 12.5 不合格服务的处置

# 12.5.1 不合格服务识别依据：

（1）质量管理体系文件；  
（2）设计工作手册；  
（3）与服务有关的其它管理文件；  
（4）顾客的评价。

# 12.5.2 不合格服务的预防和处置

（1）各组织必须提高质量意识，明确工作职责，以防止不合格服务的发生。  
（2）在设计过程中发生不合格服务时，设计人员应立即终止服务，并采取补

救或改正措施，减少负面影响，尽量使服务活动规范和满足业主或顾客需求。

（3）在产品交付后，发生不合格服务时，所在组织负责人应对不合格服务及  
造成的影响采取补救或纠正措施，情况严重的需报一号位（总经理）。  
（4）因客观条件（如天气、交通、市政等）的影响和制约，造成不能满足顾客  
要求的服务，可事先与顾客进行协调，经顾客同意后，再约定时间处理。可能时，

向顾客作出解释。

# 13 设计质量管理要求

# 13.1 设计质量控制

13.1.1设计质量控制应执行《设计质量控制管理办法》、《工程设计质量红黄线管理办法》等相关要求。  
13.1.2各级设计人员负责设计产品设计过程的质量管控，并对设计文件的质量负责，各专业审核/审定人员对本专业总体质量负责。  
13.1.3设计经理组织各设计人员根据设计过程管理相关制度及其他相关要求对项目实施管控，对项目总体质量负责。  
13.1.4各业务组织负责组织相关设计产品的质量检查，收集质量问题、获取质量信息、管控设计产品质量。  
13.1.5技术质量智能能力群负责组织分公司级的设计产品质量检查，收集质量问题、获取质量信息、统计产品质量目标数据、监测设计产品质量。  
13.1.6总工程师、技术委员会委员、各专业召集人、专业总工、设计经理及具备审核、审定设计资格人员按要求开展质量检查、评审、评定等质量管控工作，并对检查、验证等结果负责。

# 13.2 质量控制管理范围

（1）成品质量检查：季度检查、专项检查、归档检查等；  
（2）质量过程管控：事前、事中、事后；  
（3）质量问题的评审和反馈；  
（4）质量信息的收集与应用。

# 13.3 设计质量管理

13.3.1 设计质量管理应按公司《质量管理规定》执行。  
13.3.2根据组织建立本单位质量管理体系文件、实施及维护，并完善相关制度。  
13.3.3按公司体系审核计划，实施内审、外审、管理评审等质量管理体系的运维工作。

# 13.4 质量风险识别与管控

各组织应全面识别、分析、评估本组织在生产和服务过程中存在的质量风险制定应对措施以降低或消除风险，执行公司《质量风险管理规定》。

# 13.5 质量事故

质量事故的分级、调查、责任追究等执行公司《质量事故管理规定》。

# 14 纠正和改进措施实施管理要求

14.1技术质量智能能力群为质量纠正和改进措施的归口管理部门，负责纠正措施的组织实施、跟踪和验证。

# 14.2 制定及实施纠正措施

14.2.1在原因分析的基础上，从人员、使用设备、选用工艺（或方法）、环境等方面评价已制定的确保不再发生不合格产品的措施原因，制定消除不合格产生原因的措施，并填写《纠正（预防）措施表》。

14.2.2根据不合格发生的范围和对质量管理体系运行的影响程度，纠正措施可以分为两类：A类纠正措施，B类纠正措施。

# 14.2.3 A 类纠正措施

（1）经营和产品实现过程中发现的影响或涉及多个部门或项目的不合格、管理者代表认定的重大事项、重大质量事故和管理评审输出中所要采取的纠正措施均为A类纠正措施。

（2）A类纠正措施由技术质量智能能力群负责组织实施，对所实施的纠正措施结果进行跟踪和验证，《纠正（预防）措施表》，报管理者代表审阅。

# 14.2.4 B 类纠正措施

（1）质量管理体系运行过程中发现的部门或项目中的局部不合格品、内审时发现的一般不合格项，这些不符合只涉及某个部门或某个环节，针对此类不合格所采取的纠正措施为 B 类纠正措施。  
（2）B类纠正措施由业务执行部门对其原因进行分析，确定是否需要制定纠正措施，报技术质量智能能力群批准，技术质量智能能力群对所实施的纠正措施结果进行跟踪和验证。  
（3）纠正措施实施情况报告由技术质量智能能力群负责整理、备案。

# 14.3 纠正措施的评审

技术质量智能能力群组织相关人员对纠正措施实施的有效性进行评审。

# 14.4 持续改进措施的制定依据

通过收集以下方面的信息，制定持续改进措施并予以实施，持续改进质量管理体系的有效性，提高体系运作效率，增强顾客满意度。

（1）质量方针落实和质量目标完成的结果；  
（2）内部审核、管理评审的结果；  
（3）产品质量检查的结果；  
（4）顾客投诉；

（5）顾客满意度调查结果；  
（6）其他信息反馈；  
（7）数据分析结果；  
（8）公司员工提出的合理化建议。

# 14.5 持续改进措施的实施

技术质量智能能力群收集持续改进建议后，负责组织制定持续改进措施，提交管理者代表审核批准后实施（对不能确定的提交管理评审），并对其有效性进行评价。

持续改进措施的内容包括：

（1）加强人员技术培训，提高人员素质；  
（2）明确专业、单位之间或工序之间的职责分工和衔接；  
（3）设计工作手册；  
（4）修改或补充公司质量管理体系和有关技术管理文件；  
（5）其它可能采取的措施。
