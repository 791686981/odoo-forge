# 评审清单

顾问评审时的通用检查项。按维度组织，逐条过 BRD。

---

## 完整性检查

- [ ] 每个核心流程都有明确的开始和结束条件
- [ ] 每个审批环节都有：审批人、审批标准、驳回处理、超时处理
- [ ] 异常分支都有覆盖（退回、撤销、作废、超时）
- [ ] 并发场景有考虑（两人同时操作同一条数据）
- [ ] 权限控制覆盖了所有角色的读/写/审批
- [ ] 数据的创建、修改、删除、归档都有提及
- [ ] 报表和统计需求有提及（哪怕是"暂不需要"）
- [ ] 通知/提醒机制有考虑（谁在什么时候收到什么通知）
- [ ] 移动端/远程访问需求有明确说明

## 流程合理性检查

- [ ] 没有冗余的审批环节（同一件事不需要审两次）
- [ ] 审批链路不超过 4 层（超过了要质疑是否必要）
- [ ] 没有单点瓶颈（一个人审所有东西）
- [ ] 信息不需要手动重复录入（上一步的输出是下一步的输入）
- [ ] 每个流程都有闭环（有开始就有结束）
- [ ] 回退逻辑清晰（退回哪一步、数据怎么处理）

## 定制化检查

- [ ] 标注了每个需求的定制等级（标准 / 轻定制 / 重定制）
- [ ] 每个 🔴 重定制项都有明确理由
- [ ] 确认过"调整业务流程适应标准功能"的可能性
- [ ] 对比过行业同类做法，不是想当然的"我们不一样"

## 行业对标检查

- [ ] 对比了同类公司/同行业项目的做法
- [ ] 行业最佳实践有引用或说明
- [ ] 项目未来的业务扩展方向有考虑
- [ ] 行业趋势（数字化、智能化）有提及

## 合规 & 风险检查

- [ ] 安全生产相关的审批/会签覆盖了
- [ ] 资质管理（人员资质、企业资质）有考虑
- [ ] 审计留痕要求有覆盖（操作记录、修改历史）
- [ ] 数据安全和访问控制有明确要求
- [ ] 电子签章/电子签名的法律效力有考虑（如果涉及）
- [ ] 档案管理要求有覆盖（设计文件归档、保密等级）

## 跨域检查

- [ ] 跨域数据依赖明确（谁给谁、什么数据、什么时候）
- [ ] 跨域流程的触发和联动机制清楚
- [ ] 域间数据一致性有保障（不会两边数据对不上）
- [ ] 跨域的权限边界清晰

## 可扩展性检查

- [ ] 没有写死的业务假设（比如"只有 3 个部门"）
- [ ] 数据模型能适应未来增长
- [ ] 项目未来业务扩展方向的路径没有被堵死
- [ ] 多项目/多组织的场景有预留
