# 项目设备材料请购控制程序

# 1. 范围

本标准规定了项目设备、项目材料的请购工作程序。

本标准适用于EPC工程建设项目的设备/材料请购工作，其他业务组织、区域公司根据项目实际情况可参照执行。

# 2. 规范性引用文件

下列文件中的条款通过本标准的引用而成为本标准的条款。凡是注日期的引用文件，其随后所有的修改单（不包括勘误的内容）或修订版均不适用于本标准。凡是不注日期的引用文件，其最新版本适用于本标准。

ENN NG 9920/MAT001A-2022 建造智能业务群物资管理制度

# 3. 职责

# 3.1 设计组职责

设计人员负责编制《请购文件》，并对所请购设备/材料的数量及要求负责。

编制人、校核人、审核人和设计经理对《请购文件》的准确性与完整性负责。

# 3.2 项目组职责

项目组内材料控制工程师审核确认设计需要量、《设备/材料表》是否满足系统导入要求及确认设计部门是否给出关键等级；

计划工程师确认设计是否给出厂商资料需求，同时提出计划到场时间；

费用控制工程师确定控制价格；

文档控制工程师审核文件编号是否有误；

采购主管（项目级）审核待批准的《请购文件》深度是否满足采购的要求，《设备/材料表》中的打包分类是否满足采购需求。对于升版的《请购文件》，采购主管还应核实并返回上版采购状态。

采购经理负责接收正式《请购文件》并严格按照其中规定的设备/材料数量及要求进行采购。

项目组所有审核人员对自己的审核内容负责。

项目经理对《请购文件》整体完整性负责。

# 3.3 系统运维组职责

系统运维组负责材料控制与采购管理系统与EPC系统的技术支持工作。

# 4. 请购程序

请购程序包括《请购文件》的编制、审核与发布。其中《请购文件》的编制分为线上和线下两种。设备材料请购流程详见附件3 《请购文件发送流程》。

# 4.1 《请购文件》的编制

# 4.1.1 编制责任人

设备/材料请购文件由各相关专业设计人员负责编制、校对和审核，设计经理审批。

# 4.1.2 请购量

设备请购量应是该位号下运行设备和备用设备数量之和。材料请购量应是设计数量和裕量之和。材料裕量根据公司有关规定进行测算。

# 4.1.3《请购文件》提出时间

设计人员应根据项目进度计划的要求，分批提出请购文件。对长周期、关键设备/材料应尽早提出请购文件，以保证采购周期。最终料表完成后，编制最终的请购文件。

# 4.1.4《请购文件》的组成

一套完整的《请购文件》由《设备/材料请购文件》（模板见附件1）、《设备/材料表》（模板见附件2）组成。

《设备/材料请购文件》包括：请购文件封面、目录、会签页（如有）及其他成品文件。其他成品文件包括但不限于：询价书、规格书、数据表、材料表、各类图纸、统一规定等。所有成品文件均使用原有编号纳入目录中。编制、校对和审核在请购文件封面签字时需同时审核整份请购文件的完整性。在请购文件规格书中，要对设备/材料的规格、材质、技术要求、制造标准等进行明确规定，使采购组能按要求进行采购，保证所购设备/材料满足设计要求。

# 4.2 《请购文件》的编码

《请购文件》编码依各项目设计统一执行。原则上应包括项目名称、专业代号、请购文件标识码等信息。

# 4.3 《请购文件》的审核程序

# 4.3.1设计组审核和批准

《设备/材料请购文件》编制完成后，首先由该专业的校核人、审核人（或专业负责人）进行审核。

经校核人、审核人审核通过后将《设备/材料请购文件》提交设计经理批准，如同意，则进入下一程序。如不同意，则提出意见并退回编制人。

# 4.3.2 设计组提交

《设备/材料请购文件》通过设计组批准后，请购人编制《设备/材料表》和《审批单》，纸版《审批单》经设计经理签字后提交项目组审核。提交文件内容及文件传递方式等要求见表1 提交文件清单：

# 4.3.3项目组审核和批准及时间要求

项目组收到设计组提交的《请购文件》后，相关人员进行审核并完成《审批单》的签字传递工作，将《审批单》提交至项目经理。如有需要设计修改的内容，设计应按照审批意见修改并重新提交。项目组审核应在2个工作日内完成。

# 4.3.4项目组审核内容

材料控制工程师在邮件接收《请购文件》后，首先审核确认请购数量。材料控制工程师应在综合项目库存物资的基础上，根据设计需要量进行审核确认，如确认所有请购量，则在《审批单》上签字，如不同意请购量，应提出审核意见，并返回请购人。

材料控制工程师在确认请购量后，应将《设备/材料表》导入材料控制与采购管理系统（MCPMS）。如无法正常导入，则返回请购人修改。经材料控制工程师确认后，《审批单》提交至计划工程师。计划工程师审核设计是否给出设备/材料的厂商资料需求。同时参考项目进度计划及现场情况，提出计划到达现场日期。

费用控制工程师在邮件接收《请购文件》后提出估算单价和总价，采购组的采购价格一般不应超过费用控制工程师规定的价格限额。如无异议，则在《审批单》上签字，如有严重违背概算，应提出审核意见，并返回请购人。

文档控制工程师在邮件接收《请购文件》后核实文件编号是否有重复、错乱、编写错误等问题，如有问题应及时回复请购人修改。

采购主管在邮件接收待批准的《请购文件》后，负责审核请购内容（含描述）是否满足采购要求；同时，采购主管应审核《设备/材料表》中的包号是否满足订单需求。如任何一点不满足要求应及时回复请购人修改。对于升版的《请购文件》，采购主管需在邮件中返回上版采购状态。

采购组应按审批完成的正式请购单核准数量组织采购。

# 4.3.5项目经理批准

完成上述工作后，材料控制工程师将纸质签字版《设备/材料请购文件》及《审批单》提交至文档控制工程师，由文档控制工程师提交项目经理批准。

# 4.3.6《请购文件》发布

经项目经理批准后，文档控制工程师将《审批单》扫描并将完整的《请购文件》(电子版)发布给采购经理，抄送所有审核人员，同时将《请购文件》（纸质签字版）提交至采购经理。文档控制工程师正式发布《请购文件》(电子版)的时间即视为采购部接收《请购文件》的时间。

采购经理在收到《请购文件》(电子版)后应尽快向文档控制工程师发送回执。文档控制工程师在24小时内未收到回执则应电话联系采购经理确认。

单个装置的请购由项目采购经理负责协调专业组制定专业采购人员进行采购，框架（集采）或框架装置打包的请购由项目采购主任负责安排专业组进行采购。采购综合计划负责接收和分发请购，并负责建立请购台账进行相应跟踪追溯。

# 4.4 《请购文件》的升版

已发布的《请购文件》如需修改（含作废），则必须升版。升版的《请购文件》应按本程序重新审核及发布。采购主管收到升版《请购文件》的审核邮件时，需确认并邮件回复上版《请购文件》的采购状态，即订单签订状态。“已签订单”即为系统内材料不可更改，“未签订单”即为系统内材料可以更改。上版采购状态仅作为系统内材料是否更改的依据，不等同于实际订单的签订状态，也不影响本版《请购文件》的正常发布。

# 4.4.1升版时《审批单》的编制

升版的请购文件《审批单》需要增加升版说明及上版采购状态。如《请购文件》因变更而升版，则还应填写联系单号，联系单号为升版《请购文件》所引用或相关联的《工程联络单》或《设计变更单》编号。

# 4.4.2升版时《设备/材料请购文件》的编制

《设备/材料请购文件》包括：请购文件封面、目录、会签页（如有）及其他成品文件。其他成品文件包括但不限于：询价书、规格书、数据表、材料表、各类图纸、统一规定等。所有成品文件均使用原有编号纳入目录中。升版时只需附上有修改的成品文件（含云形线标识），无修改的成品文件在目录“备注”列标明“沿用上版”即可。材料表（带公司LOGO的成品格式）需附升版后的总材料表。

# 4.4.3升版时《设备/材料表》的编制

升版《请购文件》所附电子版《设备/材料表》（POWERON接口格式）为《差量设备/材料表》。

# 4.4.4《请购文件》的作废

《请购文件》如需作废，则升版时只需提交《审批单》（电子版及纸版）和《差量设备/材料表》（电子版）即可。

# 4.5 MCPMS 系统内《请购文件》的编制及发布

材料控制工程师将《设备/材料表》导入MCPMS系统内对应的装置节点下形成汇总BOM表，接着在该装置节点下按照包号生成《请购文件》，随后在MCPMS系统内发布。

# 4.6 MCPMS 系统内《请购文件》的升版

当《请购文件》升版时，材料控制工程师应参考升版《请购文件》中的“上版采购状态”导入《差量设备/材料表》。

导入材料表后对线上《请购文件》进行升版及发布，使之与线下《请购文件》保持一致。

# 4.7 MCPMS 系统内《请购文件》的拆分

MCPMS系统内《请购文件》按照《设备/材料表》中的“包号”拆分。当已发布的《请购文件》需要重新拆分时，由采购主管提供《包号变更明细表》给相关材控工程师，材控工程师在系统内重新拆分。

# 5. 现场请购程序

项目现场发生的零星材料采购参照标准 ENN NG 9920/MAT005A-2022《建造智能业务群物资采购管理办法》执行。原则上经现场设计代表、设计总代表、现场材控、现场采购经理和现场经理签字后生效。
