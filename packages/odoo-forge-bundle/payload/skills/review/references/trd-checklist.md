# TRD 审查清单

## 必须检查

- 模块规划是否清楚
- 模型定义是否清楚
- 权限设计是否清楚
- 视图实现思路是否清楚
- 数据文件、初始化数据、配置数据是否交代
- 测试指引是否交代
- 与 PRD、Solution 是否一致

## 高价值检查

- 开发拿到后能否直接开始实现
- 是否存在模型、字段、方法职责不清
- 是否存在权限和业务规则冲突
- 是否存在视图实现和业务规则冲突
- 是否遗漏关键依赖或技术前提

## Odoo 适配检查

- 模型设计是否顺 Odoo 标准模式
- 是否存在明显违反官方视图 / 权限 / ORM 常识的地方
- 继承、扩展、定制边界是否合理

## 红旗信号

- 只有模块名，没有模型细节
- 只有模型，没有行为边界
- 只有技术实现，没有回扣 PRD 规则
- 视图实现只写“按标准实现”，没有关键细节
- 测试指引过于空泛

## 适合加入页面《待确认事项》的典型问题

- 某个技术实现依赖的业务口径未定
- 某个权限边界是否允许例外未定
- 某个视图行为是否需要额外规则支持未定
- 某个官方能力边界是否足以覆盖当前实现未定
