# PRD 审查清单

## 必须检查

- 用户故事是否覆盖核心角色和目标
- 业务规则是否明确
- 状态机是否闭环
- 权限边界是否清楚
- 视图、菜单、入口是否清楚
- 异常流、边界条件是否覆盖
- 与 BRD、Solution 是否一致

## 高价值检查

- TRD 拿到后是否还需要猜业务逻辑
- 是否存在模糊词：支持、适配、灵活、可配置，但没有具体定义
- 是否存在状态、按钮、规则之间对不上的情况
- 是否缺少影响交互的重要前置条件和后置结果

## 交付就绪检查

- 规则是否细到可以被技术设计接住
- 页面行为是否清楚到可以继续下钻视图规格
- 权限与流程的对应关系是否成立

## 红旗信号

- 只有流程，没有规则
- 只有字段，没有行为
- 只有 happy path，没有异常流
- 视图描述和状态规则互相打架
- “由系统判断”“按规则处理”但没有说规则是什么

## 适合加入页面《待确认事项》的典型问题

- 某个状态转换的真实触发条件未定
- 某个角色是否应该有例外权限未定
- 某个页面行为是否要走更简还是更严的流程未定
- 某个业务规则的优先级或冲突处理未定
